# Flash-Hub (优选社交点评系统)

这是一个基于 Spring Boot 2.x 的高性能社交后端系统。项目核心攻克了高并发场景下的秒杀安全、消息队列全链路可靠性、多级缓存一致性以及社交 Feed 流分发等技术难点。

## 🌟 技术栈
- **核心框架**: Spring Boot 2.3.12, MyBatis Plus 3.4.3
- **消息中间件**: **RabbitMQ** (集成 Publisher Confirm, Lazy Queue)
- **数据存储**: MySQL 5.1.47, Redis 5.x (StringRedisTemplate, Redisson 3.13.6)
- **核心工具**: Hutool, Lombok

## 🚀 核心功能与技术实现

### 1. 高并发秒杀模块 (Redis + RabbitMQ 深度集成)
该模块实现了从“同步阻塞”到“异步削峰”的架构演进，确保了极端并发下的数据一致性：
- **原子性操作**: 编写 **Lua 脚本** 挂载至 Redis，实现“库存校验-一人一单判断-预扣库存”逻辑，确保在 Redis 层完成原子执行，有效拦截 90% 的无效请求。
- **RabbitMQ 削峰填谷**: 引入消息队列异步处理订单入库，将原本沉重的数据库写操作异步化，极大提升了接口响应速度并保护了后端数据库。
- **全链路可靠性保障**:
    - **生产者可靠性**: 开启 `publisher-confirm` 机制并配合连接重试，确保消息 100% 到达交换机。
    - **高性能存储**: 队列开启 **Lazy Queue (惰性队列)** 模式，支持百万级消息在磁盘堆积，防御瞬时流量冲击导致的内存溢出。
    - **消费者保障**: 采用 `auto` 确认模式配合**本地重试机制**。在重试耗尽后利用 `RepublishMessageRecoverer` 将消息转发至 **Error Queue** 记录，实现异常可追溯。
- **分布式锁二次校验**: 消费者端集成 **Redisson** 分布式锁，在异步入库阶段进行二次校验，彻底杜绝集群环境下的超卖与重复下单问题。

### 2. 企业级缓存实战 (高性能查询)
项目针对高并发访问压力，构建了多层防护体系，由 `CacheClient` 工具类统一封装：
- **缓存击穿与异步重建**: 
    - **逻辑过期机制**: 实现 **逻辑过期 (Logical Expire)** 方案，在 Redis 存储中封装自定义 `RedisData` 对象，包含业务数据及其逻辑过期时间。
    - **零延迟响应**: 当检测到数据逻辑过期时，利用自定义线程池 `CACHE_REBUILD_EXECUTOR` 开启异步重建任务，主线程立即返回“旧数据”，实现热点数据查询的零阻塞。
- **冷启动保护**: 针对缓存不存在的“冷数据”，首个请求同步查询数据库并返回，同时利用异步线程池快速补齐带逻辑过期的缓存记录，平滑度过冷启动阶段，杜绝数据库穿透。
- **缓存穿透防护**: 采用缓存空对象策略，在数据库查询无果时向 Redis 写入短 TTL 的空字符串 `""`，有效防御恶意请求对数据库的直接冲击。

### 3. 社交 Feed 流 (推拉模式研究)
- **高效分发**: 实现 **推模式 (Push)** 的关注博文分发，利用 Redis **ZSet** 数据结构按照时间戳排序，构建高性能用户收件箱。
- **共同关注**: 基于 Redis **Set** 集合的交集运算实现好友关联推荐。

## 📂 项目结构亮点
- `src/main/resources/*.lua`: 核心业务的原子性保障脚本。
- `src/main/java/com/hmdp/config`: 包含 `RabbitMqConfig` 和 `ErrorConfig` 等可靠性保障核心配置。
- `src/main/java/com/hmdp/service/impl`: 采用 **循环依赖注入自身代理** 模式（`voucherOrderService`），完美解决异步环境下 AOP 代理失效导致的事务与分布式锁失效问题。
